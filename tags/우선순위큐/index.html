변화하는 중간 값(RUNNINGMEDIAN)

변화하는 중간 값(RUNNINGMEDIAN) (난이도:下)12 내 풀이 N(1≤N≤200,000) 개의 입력 => O(N·logN) 으로 구현해야 함 push_heap 해주고 i/2 가운데 값 더해줌 문제점 일단 힙은 루트에만 가장 큰 값이 올라가고 나머지는 정렬x 틀렸다는 것 찾았지만 아무리 생각해도 logN 만에 중간값 찾는 방법 생각 못 함

우선운위큐와 힙

우선순위 큐와 힙 우선순위 큐 : 순서대로 기다리고 있는 자료들을 저장하여 우선순위가 가장 높은 자료를 가장 먼저 꺼내주는 자료 구조 우선순위 큐의 구현연결리스트, 배열 : 원소를 꺼낼 때마다 모든 원소를 순회하며 우선순위가 가장 높은 원소를 찾는다. ​ 원소 추가에 O(1), 원소 꺼내기에 O(N)
