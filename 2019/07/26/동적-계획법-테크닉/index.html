동적 계획법 테크닉

k번째 답 계산하기

모스 부호 사전 (MORSE)
  1. 모든 신호 만들기 : 완전 탐색으로 skip을 줄여가면서 k 번째 답을 출력
  2. 건너뛰기 : ‘-‘이 n개, ‘o’가 m개라고 할 때, 이들을 조합해 만들 수 있는 부호는 총 (n+m)!÷n!m!개이다. 만약 skip이 이보다 크거나 같다면 그 개수만큼 skip을 줄여버리고 재귀 호출하지 않고 return 시킨다.
  3. 시작점을 기준으로 잡고 이어나가기. (-,o)의 개수가 (n,m)이라고 할 때 ‘-‘로 시작하는 조합의 개수는 (n-1,m)개. 따라서 skip이 이 개수보다 크다면 무조건 ‘o’로 시작. 작다면 ‘-‘로 시작한다. 이 과정을 반복

정수 이외의 입력에 대한 메모이제이션

연관 배열 이용하기 : map<vector<int˙>,int> cache;

vector를 키로 갖는 map을 이용할 수 있다. 그러나 map에 접근하는데 시간이 걸리고 vector와 같은 컨테이너들은 비교하는 데도 시간이 오래 걸리기 때문에 계산량이 작을 때만 사용할 수 있다.

일대일 대응 함수 작성하기

입력의 값을 조절하여 정수 형태로 일대일 대응시켜 메모이제이션을 적용해보자.

  1. 입력이 불린 값의 배열인 경우 : {true} : 1 = 1, {true, false, true, false} : 0101 = 5 같이 변형 가능
  2. 순열인 경우 : 사전순으로 몇 번째 오는지를 기준으로 정수로 변환
여행하는 외판원 문제

원래 완전 탐색을 이용해 모든 경로에 대한 크기를 구해줘서 비교하는 문제였다. 같은 인자를 갖는 함수가 다시 호출되지 않아서 메모이제이션을 쓸 수 없다. 메모이제이션을 적용할 수 있도록 visited 배열을 정수 형태로 변환해보자. 각 지점을 방문했으면 1, 방문하지 않았으면 0으로 지정, 즉 2진수로 표현해줄 수 있다. visited는 다음과 같이 표현 가능하다.

모두 방문했을 경우 : visited==1<<n-1
방문했는지 확인 : visited&(1<<next)==1
next를 방문하여 visited에 추가 : visited+=(1<<next)

이렇게 정수 형태로 변환한 후 cache[here]˙[visited]로 메모이제이션을 적용해줄 수 있다.
