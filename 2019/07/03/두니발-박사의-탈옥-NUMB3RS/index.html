2019-07-03
Memoization, 다이나믹, 중

두니발 박사의 탈옥(NUMB3RS) (난이도:中)

내 풀이
  • getPercent(now, day) : 두니발 박사의 현재 위치, 남은 일수
  • if(!day) -> now==Q?1:0 : d일이 다 지났을 때 박사의 현재 위치가 Q이면 1 반환
  • ret의 기본값은 0으로 부여
  • now가 갈 수 있는 도시를 탐색하여 재귀 함수를 호출. 결과값의 합을 갈 수 있는 도시의 개수로 나눠줌
문제점
  • cache[n].[d] 인데 [n].[n] 으로 초기화해서 찾느라 오래 걸림…
또 다른 풀이
  • 계산의 순서를 거꾸로 바꿈 -> 도시 Q에서부터 d일 째에 p에 갈 수 있는지
  • 장점 : 도착 지점이 고정돼 있어서 찾고자하는 도시 입력 값이 바뀌어도 재귀 함수의 입력 값만 바뀌고 기저 사례는 동일 -> cache 재사용 가능 -> 시간 복잡도 감소
시간 복잡도

​ (부분 문제의 개수) × (한 부분 문제를 풀 때 필요한 반복문의 수행 횟수)

  1. 정방향 : ( nd × n ) × 찾고자 하는 도시 t개 = O(n^2·t)
  2. 역방향 : ( nd × n ) = O(n^2)
