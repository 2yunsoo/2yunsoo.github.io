이진 검색 트리

검색 트리 : 자료들을 일정한 순서에 따라 정렬한 상태로 저장해 두는 컨테이너

이진 검색 트리란?

각 노드가 왼쪽과 오른쪽, 최대 두개의 자식 노드만을 가지며 왼쪽 자손은 모두 부모 노드보다 작은 값을 가지고 오른쪽 자손은 모두 부모 노드보다 큰 값을 가지는 트리

순회 : 이진 검색 트리를 중위 순회하면 크기 순서로 정렬된 원소의 목록을 얻을 수 있다.

검색 : 트리 내의 비교 중인 노드보다 작으면 왼쪽, 크면 오른쪽 절반만 보면 되기 때문에 전체 대상의 수를 매번 절반씩 줄일 수 있다.

추가 : 선형적인 구조의 제약이 없기 때문에 새 원소가 들어갈 위치를 찾고 그냥 노드를 추가해준다.

삭제 : 삭제하려는 노드를 A, 왼쪽 서브트리를 B, 오른쪽 서브트리를 C라고 하자. 이 때, A가

  1. 단말 노드인 경우 : 그냥 삭제한다.

  2. 한 개의 서브트리만을 가지는 경우 : A는 삭제하고 B, 또는 C를 A의 부모 노드에 붙여줌

  3. 두 개의 서브트리를 가지는 경우 :

    B를 루트로 하는 왼쪽 서브트리만 떼어내 A의 부모 노드에 붙여주고 B의 오른쪽에는 A의 오른쪽 서브트리를 붙여준다. 그러고 오른쪽 서브트리 내에서 가장 왼쪽의 노드에 처음 B의 오른쪽 서브트리를 붙여준다.

이진 검색 트리의 시간 복잡도

이진 검색 트리는 재귀 호출을 통해 루트에서부터 노드를 타고 내려가기 때문에 높이가 h인 트리의 경우, 연산의 시간 복잡도는 O(h)이다. N개의 노드가 있을 때, 최악의 경우는 O(N-1), 최선의 경우는 O(logN)이 된다.


표준 라이브러리의 이진 검색 트리는 대부분 X보다 작은 원소의 수를 계산하거나 k번째 원소를 찾는 연산을 지원하지 않는다. 따라서 직접 구현 해야 하고 이 때 균형 잡힌 이진 검색으로 구현해야 한다. 트립은 대표적인 균형 잡힌 이진 검색 중 하나이다.

트립(treap)

부모의 우선순위가 자식의 우선순위보다 높도록 난수를 통해 생성된 우선순위를 부여하여 랜덤화된 이진 검색 트리이다. 때문에 원소들이 어느 순서대로 추가되고 삭제되더라도 트리 높이의 기대치는 항상 일정하다. 트립은 이진 검색 트리의 조건과 모든 노드의 우선순위가 각자의 자식 노드보다 크거나 같은 힙의 조건을 동시에 가지고 있다.

구현법

  • 우선 순위 priority를 가짐
  • left, right가 바뀔 때마다 자동으로 갱신되는 size 가짐
  • size를 사용해 k번째 원소를 찾는 연산이나 X보다 작은 원소를 세는 연산 등을 쉽게 구현 가능
