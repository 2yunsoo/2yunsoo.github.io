삽입 정렬 뒤집기 문제(INSERTION) (난이도:中)

내 풀이
  • n의 범위가 50000까지기 때문에 시간복잡도는 O(n·logn)
  • n개의 입력을 스캔하면서 정답을 구해야 하므로 입력에 대한 결과값을 구하는 시간은 logn -> 트리 구조를 사용
  • 각 경우에 대해 값들이 이동한 칸의 수는 트리에서의 노드 수를 확인
문제점
  • 트리의 노드 수를 확인하면 size 값을 가지고 있는 트립이 딱이지만…
  • 문제에서 설마 그 트립을 구현해서 사용하라고 할까? 싶어 MAP을 이용한 풀이를 열심히 생각
올바른 풀이
  • 뒤에서부터 보면서 n-a[n]이 원래 n번째의 값
  • k번째 원소가 무엇인지를 찾고, 삭제하는 작업을 구현하는 문제이므로
  • 그냥 트립 사용
  • 기본값으로 1~n으로 채워진 트립을 생성하고
  • ( n-a[n]번째 원소를 찾아 빼주면 해당 노드 삭제하고 n 값은 1 줄여줌 ) 반복
  • 트리에서의 탐색과 삭제는 O(logn), 전체는 O(n·logn)
느낀 점
  • 시간복잡도는 당연한 거긴 하지만 잘 접근했으나
  • k번째 원소를 찾는 문제라는 생각을 못했다.
