변화하는 중간 값(RUNNINGMEDIAN)

2019-07-10
우선순위큐, 하, 힙

변화하는 중간 값(RUNNINGMEDIAN) (난이도:下)

내 풀이
  • N(1≤N≤200,000) 개의 입력 => O(N·logN) 으로 구현해야 함
  • push_heap 해주고 i/2 가운데 값 더해줌
문제점
  • 일단 힙은 루트에만 가장 큰 값이 올라가고 나머지는 정렬x
  • 틀렸다는 것 찾았지만 아무리 생각해도 logN 만에 중간값 찾는 방법 생각 못 함
올바른 풀이
  • heap 함수를 하나하나 구현하지 말고 C++ STL의 priority_queue 사용
  • 입력 수열을 절반으로 나눠 두 개의 힙에 넣어준다.
  • 정렬했을 때 작은 값들의 묶음은 max힙으로 큰 값들의 묶음은 min힙으로 만든다.
  • 이 때 두 힙은 다음의 두 가지 규칙을 만족해야 한다.
    1. max힙의 크기는 min힙의 크기와 같거나 1만큼 크다.
    2. max힙의 최댓값은 min힙의 최솟값보다 작거나 같다.
  • 새로운 수를 입력받을 때 첫 번째 규칙을 만족시키게끔 max힙과 min힙에 번갈아 넣어준다.
  • 만약 새로운 수를 push하여 2번 규칙이 깨지게 되면 두 힙의 루트를 서로 바꿔준다.
  • 끗
  • N개를 입력받는 동안 push에 logN만큼의 시간이 걸리므로 NlogN
  • 트립도 가능하지만 구현도 문제고 포인터를 따라가기 때문에 속도도 두세 배는 느리다고 한다.
느낀 점
  • 일단 우선순위큐 구현 = 힙 그 자체인 것 같다.
  • 역시 얼른 C++ 공부해야겠다.
  • 두 개 나누는 거 어떻게 생각하는걸까
  • 생각해내고나면뒤는쉽다.
  • 근데시작이안된다.
