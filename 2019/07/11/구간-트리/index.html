구간트리

구간 트리 (segment tree)

저장된 자료들을 적절히 전처리해 그들에 대한 질의들을 빠르게 대답할 때 사용하는 트리

흔히 일차원 배열의 특정 구간에 대한 질문을 계산하는데 사용

구간 트리는 주어진 배열의 구간들을 표현하는 이진 트리로 만든다. 루트는 배열의 전체 구간 [0,n-1]을 표현하며, 각 자식들은 해당 구간의 절반, 리프 노드는 길이가 1인 구간을 표현하게 된다. 이 트리의 노드마다 해당 구간에 대한 계산 결과를 저장해 두어 전처리를 해놓는다. 전처리를 해놓으면 어떤 구간 [i,j]가 들어오더라도 구간 트리에 포함된 노드들의 구간의 합집합으로 표현이 가능해진다. 따라서 시간복잡도는 O(logN) 이 된다.

구간 최소 쿼리 (Range Minimum Query, RMQ)

구간 트리의 표현

구간트리는 비교적 ‘꽉 찬’ 이진 트리이기 때문에 이진 검색 트리처럼 포인터로 연결된 객채보다는 배열로 표현하는 것이 메모리 측면에서 더 효율적이다. 배열의 길이는 정확하게는 n보다 큰 2의 거듭제곱 × 2 또는 편하게 4n으로 잡을 수 있다.

구간 트리의 초기화

각 노드를 찾아가는 과정에서 표현하는 구간을 동적으로 계산할 수 있기 때문에 노드의 위치를 따로 저장하지 않는다. 초기화 과정의 시간 복잡도는 O(n)이 된다.

구간 트리의 질의 처리

query(left,right,node,nodeLeft,nodeRight) = node가 표현하는 범위 [nodeLeft,nodeRight]와 최소치를 찾고자 하는 구간 [left, right]의 교집합의 최소치를 반환한다.

  • 교집합이 공집합인 경우 : 두 구간이 서로 겹치지 않으므로 MAX 값으로 반환해준다.
  • 교집합이 [nodeLeft, nodeRight]인 경우 : [left, right]가 노드의 구간을 완전히 포함하므로 노드에 저장된 최소치를 반환해준다.
  • 이 외의 경우 : 두 개의 자손 노드에 대해 query()를 재귀 호출한 뒤, 두 값 중 더 작은 값으로 반환해준다.
구간 트리의 갱신

예제 : 특정 구간에서 최소치 두 개 찾기

각 구간에 대해 가장 작은 두 개의 값을 저장해 둔다. 두 구간을 합치는 경우에는 각 두 개씩, 총 4개의 최소치 중에 작은 두 개의 값을 반환하면 된다. 구간의 길이가 1일 때는 해당 구간의 숫자와 ∞을 최소치로 저장해준다.

예제 : 정렬된 수열의 특정 구간에서 최대 출현 빈도 계산

정렬된 정수 수열 A[]가 주어질 때, 주어진 구간의 최대 출현 빈도를 계산하고자 한다.

단순히 각 구간에 대해 최대 출현 빈도만을 저장하고 두 구간을 합칠 시 보다 큰 값을 반환할 경우, 다음의 세 가지 예외 사항이 발생한다.

  1. 두 구간에서 최대로 출현한 값이 같을 경우 : 두 구간의 최대 출현 빈도를 더해야 한다.
  2. 한 구간의 최대 출현 빈도를 가지는 값이 다른 구간에 존재하는 경우 : 추가된 값의 빈도를 더해서 더 큰 경우 바꿔줘야 한다.
  3. 두 구간을 합쳐 최대로 출현한 값이 바뀔 경우 : 새로 구해줘야 한다.

입력이 정렬된 수열로 들어오기 때문에 위의 세 예외 사항은 모두 왼쪽 구간의 오른쪽 끝 값과 오른쪽 구간의 왼쪽 끝 값이 같아 해당 값의 출현 빈도가 늘어날 때 발생함을 알 수 있다. 따라서 구간 트리 생성 시, 왼쪽 구간의 오른쪽 끝 수 및 출현 빈도, 오른쪽 구간의 왼쪽 끝 수 밑 출현 빈도를 추가로 저장하면 된다.
