실험 데이터 복구하기(RESTORE)

2019-07-31
다이나믹, 메모이제이션, 중

실험 데이터 복구하기(RESTORE) (난이도:中)

내 풀이

완전 탐색을 돌리면 15!만큼 걸린다. 여기다가 메모이제이션을 적용하자.

메모이제이션을 위해 우선 해당 문자열을 사용했는지 여부를 표현하는 변수 taken을 만든다. 또한 새로운 문자열을 추가할 때는 맨 뒤의 문자열과 비교하므로 가장 마지막으로 추가한 문자열이 무엇인지 나타내는 변수 last를 사용한다. 입력 문자열들 중 다른 문자열에 포함되는 문자열은 의미가 없으므로 삭제해버린다.

재귀 함수 내에서 반복문을 돌면서 taken을 확인해 아직 해당 문자열이 사용되지 않았다면 last 문자열과 비교를 해서 겹치는 문자 개수를 찾아 추가하는 문자열 개수에서 그 개수만큼 빼고 전체 크기에 더해준다.

요런 식으로 재귀 함수를 구현해주면 끝.

문제점

뭔 생각인지 문자열 최소 길이를 구하고 있었음.

최소 길이의 문자열을 저장하려고 온갖 고민을 다 해봤는데 생각나지 않음

올바른 풀이

기본 접근은 같다. 마지막 문자열 last와 taken을 인자로 사용.

last와 추가하는 문자열 사이에 겹치는 최대 문자 개수를 그 때 그 때 구하면 시간이 오래 걸리므로미리 싸 다 구해서 배열에 넣어놓고 시작.

재귀 함수가 문자열의 최소 길이를 구하는 것이 아니라 겹치는 문자의 최대 개수를 구함.

최소 문자열을 저장하는 것이 아니라 저장된 cache를 이용해 직접 구해주는 것이 해답.

cache에는 각 경우마다 겹치는 문자열의 최댓값이 저장되어 있으므로 맨 앞에서부터 restore()를 돌려서 나오는 값이 최대 문자열 개수인 애를 찾아서 추가시킨다.

종만북 + 내 풀이

종만북에서처럼 겹치는 문자열의 최댓값을 구해주는 것이 좀 더 직관적이겠지만 내 풀이로도 생각해보자. 각 경우에서의 최소 문자열의 길이가 저장돼있으므로 앞에서부터 현재 문자열 + i에 대한 for문 돌리면서 restore(i,taken에 i추가) 했을 때 최소 길이와 같으면 해당 문자열을 추가. 시키는 방식으로도 바꾸면 풀 수 있을 것 같다.
