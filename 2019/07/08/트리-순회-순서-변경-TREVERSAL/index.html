트리 순회 순서 변경(TREVERSAL)

2019-07-08
트리, 하

트리 순회 순서 변경(TREVERSAL) (난이도:下)

내 풀이
  • 전위 순회에서 첫 번째 노드는 트리의 루트이다.
  • 중위 순회에서 루트 노드의 좌우는 각각 왼쪽, 오른쪽 서브트리로 나눌 수 있다.
  • 각각 중위 순회에서 (루트 노드의 위치 - 1) , (루트 노드의 위치 + 1)의 크기를 가지는 두 서브트리를 재귀함수에 다시 넣어준다.
  • start가 end의 범위를 넘어서는 경우는 더 이상 서브트리가 존재하지 않는 것
  • 후위 순회의 순서는 좌, 우, 루트이므로 루트, 우, 좌의 순서로 push해주고 역순으로 출력해준다.
문제점
  • 재귀 함수의 인자로 전위 순회와 중위 순회의 시작 위치와 끝 위치를 사용하였는데 이 때 이 값을 결정해주는 것은 각 서브트리의 크기이다. 이상하게 삥삥 돌다가 다 지우고 다시 쓰고야 알았다…
종만북 풀이
  • 재귀 함수의 인자로 위치가 아닌 서브트리 vector를 사용하였다. 내꺼가 짧다ㅎㅎ
느낀 점
  • 코드 짤 때 머릿속에서 제대로 정리 마치고 한번에 두두두 짜자. 안 그러면 쓸데 없는 거에서 헤맨다.
