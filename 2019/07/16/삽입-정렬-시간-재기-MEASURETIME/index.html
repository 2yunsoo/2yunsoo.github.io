삽입 정렬 시간 재기(MEASURETIME)

2019-07-16
구간트리, 병합정렬, 트립, 펜윅트리

삽입 정렬 시간 재기(MEASURETIME) (난이도:中)

내 풀이
  • logn에 해결봐야됨
  • 트립 사용하면 k보다 작은 노드의 개수 셀 수 있음
  • 근데 펜윅 트리 문제 연습이라 그걸로 아무리 생각해봐도 생각이 안남…

문제점
  • 음… 펜윅트리로 어떻게 표현해야할 지 몰랐음

올바른 풀이

1. 펜윅 트리

입력 데이터의 범위가 1,000,000 미만이기 때문에 일차원배열로 잡고 입력 받으면서 해당 숫자의 개수를 저장해준다. 그러고 펜윅트리를 사용해 전체 숫자의 개수에서 A[i]보다 작거나 같은 숫자의 개수를 빼주면 A[i]보다 큰 숫자의 개수를 세줄 수 있다.

입력으로 주어지는 숫자의 최댓값이 M이라고 하면 이 풀이의 시간복잡도는 O(NlogM)이다.

2. 트립

트리에 주어진 키 k보다 작은 키가 몇 개 있는지 계산하는 연산은 O(logN)이므로 O(NlogN) 내에 해결 가능하다.

3. 병합 정렬

각 입력 숫자에 대해 이미 받은 수열 내에 존재하는 더 큰 숫자를 찾는 것이 아니라 전체 수열에서 더 큰 수가 앞에 있고 작은 수는 뒤에 있는 반전의 개수를 센다. 삽입 정렬에서 숫자를 한 번 옮길 때마다 반전이 하나 줄어들기 때문에 반전의 개수와 숫자를 움직이는 개수는 같다.

반전의 수를 세는 방법으로는 병합 정렬을 사용한다. 주어진 수열을 절반으로 나눠 L, R이라고 하면 반전은

  1. 두 수가 모두 L에 포함되거나
  2. 두 수가 모두 R에 포함되거나
  3. 큰 수는 L에, 작은 수는 R에 포함되어 있다.

1, 2번은 분할 정복으로 세고 3번은 두 수열을 병합하는 과정에서 셀 수 있다. 병합하는 두 수열은 이미 정렬된 상태로 존재하기 때문에(음… 닭이 먼저인가. 달걀이 먼저인가. 정렬돼있기 때문에 이런 방법을 쓰지만 이런 방법을 쓰기 때문에 정렬돼서 존재하는 거 아닌가…) L의 앞에서부터 돌면서 만약 L에 있는 수가 R에 있는 수보다 커지면 L에 남아있는 모든 수가 R의 수보다 크다고 볼 수 있다. 따라서 L에 남아있는 개수만큼의 반전이 존재한다.
