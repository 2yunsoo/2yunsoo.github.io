2019-06-28

합친 LIS(JLIS) (난이도:下)

내 풀이
  • 두 배열을 어떻게 합쳐야 할까 고민
문제점
  • 아직 다이나믹을 이용한 최적화 문제 풀이에 익숙하지 않음
  • 결론 : 못 품…
올바른 풀이
  • 기본 구조는 LIS를 똑같이 따라감
    1. 각 배열의 시작 지점을 -1부터 넣어주고 -1일 경우에는 배열 값을 가능한 범위 내에서의 최솟값 부여
    2. 뒤의 모든 배열이 최솟값보다 크기 때문에 모든 경우를 자동으로 호출 가능
    3. 메모이제이션 적용
    4. 기저 사례 없이 기본 ret 값은 2로 부여(시작 지점이 -1일 때 두 함수의 값 두 개 존재)
    5. 출력 코드 = jlis(-1,-1) - 2 : 함수 내에서 두 배열의 시작 지점이 -1일 때에도 값을 부여하여 기본 ret 값을 2로 지정하였기 때문에 cache[]의 크기가 모두 2씩 커져 있음
  • 두 배열을 동시에 봐야하므로 함수에 두 시작지점을 인자로 부여
  • A[nowA]와 B[nowB] 중 작은 값이 앞에 오도록 설정 -> 증가 부분 수열의 다음 숫자는 max(A[nowA],B[nowB])보다 큰 숫자
다시 쓰는 풀이

2주 만에 코드를 구현하고자 해봤더니 다시 풀이를 생각해내지 못했다… 그래서 내가 쓴 걸 열어봤으나 너무 부실했다. 부연 설명을 하자면 ‘두 수열에서 구하는 합쳐진 증가부분수열은 작은 값이 앞에 오니까 그 다음 재귀 호출 시킬 수는 현재 증가 부분 수열에 들어간 A, B 두 수보다 커야 한다’ 대충 이 정도…?
